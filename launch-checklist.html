C1

Release zip wraps assets in a folder — Obsidian's review bot expects flat files

Critical
📁 .github/workflows/release.yml lines 29–33

Obsidian's review tooling (and the BRAT installer) downloads main.js, manifest.json, and styles.css directly from the release assets. Your release currently uploads a rho-reader.zip that contains a nested folder, and does not upload the three files as standalone assets.

Required

  • Attach main.js, manifest.json, and styles.css as top-level assets on the GitHub release (in addition to or instead of the zip).
  • If you keep the zip, it must contain those files at the root, not inside a rho-reader/ folder.

C2

Don't detach leaves in onunload

Critical
📁 src/main.ts:57-59

The plugin guidelines explicitly call this out as an anti-pattern: "Don't detach leaves in onunload". Obsidian handles tearing down plugin views automatically, and detaching them yourself loses the user's layout when the plugin is updated or reloaded.

Fix

-	onunload() {
-		this.app.workspace.detachLeavesOfType(VIEW_TYPE_RHO_READER);
-	}

Remove the entire onunload override (or leave it empty if you intend to add cleanup later).

C3

Command and setting names must use sentence case

Critical
📁 src/commands/register.ts, src/settings/RhoReaderSettingTab.ts

Obsidian's style guide requires sentence case for command names, setting names, and headings. Reviewers flag this on nearly every submission.

Rename

  • "Open RSS Feed Reader" → "Open RSS feed reader"  (commands/register.ts:16)
  • "Sync RSS feeds" ✅ already correct
  • "Import OPML" / "Export OPML" ✅ (OPML is an acronym)
  • Setting "RSS Feed Bases" → "RSS feed bases"  (RhoReaderSettingTab.ts:50)

Also: Obsidian no longer wants you to call addCommand with a command ID containing the plugin name — keep IDs short (current IDs are fine).

C4

minAppVersion is too old for the APIs you use

Critical
📁 manifest.json line 5, versions.json

You declare "minAppVersion": "0.15.0", but the plugin is built on top of Obsidian Bases (per your README), which only shipped in Obsidian 1.9+. A user on 0.15 would crash on the first feed open.

Fix

-	"minAppVersion": "0.15.0",
+	"minAppVersion": "1.9.0",

Update versions.json to mirror it. Reviewers verify the declared minimum matches the APIs you actually call.

Submission
